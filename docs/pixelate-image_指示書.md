## 背景
依頼者は英語圏向けAI画像ツールを多数公開し当たりを探す「多産多死ポートフォリオ戦略」の個人開発者(CV/ML研究者、12年)。本件はその**1本目**であり、目的は収益より「短期間で公開まで到達すること」。完璧さより出荷speed優先。**目標: 実装3日以内、今週中に公開。**

ターゲットキーワードは "pixelate image"(米国月間検索 約14,800、Ahrefs KD 11)。SERP調査済みで、上位は古参の寄せ集めツール(PineTools, LunaPic=2000年代のUI)、Photoshopのやり方解説記事(Adobe)、個人の単機能サイト。**個人サイトが上位に入れており、現物は古い。**勝ち筋は「初見ユーザーが10秒で結果に到達する速さ」「顔の自動検出モザイク(競合は手動選択のみ)」「完全ブラウザ内処理のプライバシー訴求」。

## プロダクト仕様(v1)

### コア体験
1. 画像を渡す: ドラッグ&ドロップ / ファイル選択 / クリップボード貼り付け(Ctrl+V)の3経路
2. 即プレビュー: 読み込んだ瞬間にデフォルト強度でピクセル化プレビューを表示(操作ゼロで最初の結果が見える)
3. 調整: ブロックサイズのスライダー(リアルタイム反映)
4. 範囲指定: 全体 / 矩形選択(ドラッグで複数可)の2モード
5. **顔自動モザイク(差別化機能):** ボタン一発で画像内の全ての顔を検出しモザイク。検出はブラウザ内で完結させる(MediaPipe Tasks Vision の Face Detector をCDNから利用する想定。実装時に最新の推奨を確認し、WASMで安定して動くものを選ぶこと。モデルのロードは顔ボタン初回押下時の遅延ロードでよい)
6. ダウンロード: 元画像と同じ解像度・選択可能な形式(PNG/JPEG/WebP)。透かしなし、制限なし

### 必須の非機能要件
- **完全クライアントサイド処理。** 画像をサーバーに一切送らない(Canvas処理)。ページ上に "Your image never leaves your device" を明記——これは競合上位も使う訴求で、かつ運用コストをゼロにする
- 静的サイト(ビルド後はHTML/CSS/JSのみ)。フレームワークは任意だが、依存は軽く。単一ページ
- モバイル(iPhone Safari / Android Chrome)で全機能が動くこと。スマホのスクショにモザイクをかける用途が多い
- 大きめ画像(20MP程度)でフリーズしないこと(必要なら内部処理は縮小プレビュー+書き出し時のみフル解像度)
- Lighthouse Performance 90以上目安。LCPを損なう重いライブラリの同期ロード禁止

### ページ構成(SEO要件)
言語は英語。1ページ内に:
- ツール本体(ファーストビュー。スクロールなしで使い始められること)
- 下部にテキストセクション: How to pixelate an image(3〜4ステップ)/ ユースケース(顔・ナンバープレート・スクショの個人情報隠し、ピクセルアート風加工)/ FAQ 5問程度(モザイクは復元できるか、アップロードされるのか、無料か、対応形式、ぼかしとの違い)
- title / meta description / OGP / FAQのJSON-LD構造化データ
- ドメインは未定のため、サイト名・URLは設定ファイル1箇所で差し替えられる構造にしておく(仮名: "PixelTools" 等でよい)

### 計測
- Cloudflare Web Analytics か Plausible(どちらも Cookie バナー不要)を1行スニペットで導入(IDは依頼者が後で差し替え。プレースホルダにしておく)
- 最低限のイベント: 画像読み込み / ダウンロード実行 / 顔モザイク使用。実装が重ければページビューのみでv1可

- サーバーサイド処理、API、DB

## 実装順
1. Canvasでのピクセル化コア(全体→矩形選択)+スライダー。まずデスクトップChromeで動くものを最短で
2. 入力3経路とダウンロード(形式・解像度保持)
3. 顔検出モザイク(遅延ロード。うまく動かないライブラリなら代替を試し、それでも不安定なら機能ごと外してv1.1に回す判断を依頼者に仰ぐ)
4. モバイル対応・パフォーマンス調整
5. LP部分(テキスト・FAQ・構造化データ・OGP)
6. デプロイ(下記)
